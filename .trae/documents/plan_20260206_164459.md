### 1. 组件库选型对比与决策

针对您的需求（<50KB、按需加载、现代化设计、无障碍支持），我对比了以下三个主流轻量级库：

| 特性 | **Spectre.css** (推荐) | **Bulma** | **Milligram** |
| :--- | :--- | :--- | :--- |
| **体积 (Gzipped)** | **~10KB** | ~22KB | **~2KB** |
| **按需加载** | **支持 (SASS)** | 支持 (SASS) | 不原生支持 |
| **设计风格** | 现代、扁平、功能丰富 | 现代、Flexbox优先 | 极简、重置为主 |
| **组件丰富度** | **高** (含按钮、模态框、表单) | 高 | 低 (仅基础排版) |
| **按钮变体** | **全覆盖** (Primary/Link/Error) | 全覆盖 | 仅基础 |

**🏆 最终选型：Spectre.css**
*   **理由：** 它的体积远低于 50KB 限制，同时提供了非常完善的组件系统（这对后续开发RPG的对话框、物品栏非常有帮助），且原生支持 SASS 模块化导入，可以轻松实现"仅引入按钮和核心样式"。其设计风格中性，易于通过覆盖变量来融合我们之前的"像素+毛玻璃"风格。

### 2. 实施计划

我们将从"手写原生CSS"升级为"基于构建工具的工程化开发"，以支持按需加载、单元测试和性能监控。

#### **Phase 1: 工程化环境搭建**
1.  初始化 `package.json` 并安装 **Vite** (构建工具)。
2.  安装 `spectre.css` (UI库) 和 `sass` (预处理器)。
3.  安装 `vitest`, `jsdom`, `@testing-library/dom` (测试框架)。
4.  安装 `lighthouse` (性能监控)。

#### **Phase 2: 按需引入与样式定制**
1.  将 `styles/main.css` 重构为 `styles/main.scss`。
2.  **按需引入：** 仅导入 Spectre 的 `Core` (重置/排版) 和 `Buttons` 模块。
3.  **样式覆盖：** 编写自定义 SCSS，将 Spectre 的按钮样式强制重写为您要求的"像素毛玻璃"风格（保留 focus ring 等无障碍特性）。
4.  **变体实现：**
    *   主按钮 (`btn-primary`): 对应"开始旅途"
    *   次要按钮 (`btn-secondary`): 对应"继续旅途"
    *   危险按钮 (`btn-error`): 用于未来的"删除存档"
    *   幽灵按钮 (`btn-link`): 用于辅助操作

#### **Phase 3: 代码集成与测试**
1.  修改 `index.html` 引入新的样式入口和 CSS 类名。
2.  编写 `tests/button.test.js` 验证按钮的点击、状态切换和 ARIA 属性。
3.  编写 `scripts/monitor-perf.js` 脚本，自动化运行 Lighthouse 检查性能分。
