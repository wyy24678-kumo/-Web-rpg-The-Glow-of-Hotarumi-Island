# 制作《萤见之岛的微光》产品需求文档 (PRD) 计划

基于您提供的世界观设定，我将为您编写一份详细的产品需求文档（PRD）。这份文档将作为后续开发的基准，重点突出“治愈系美术”与“深层情绪共鸣”的核心卖点。

## 1. 核心定位与用户分析

* **核心定位**：移动端 Web 优先的轻量级治愈系像素奇幻 RPG。

* **目标用户**：深度分析 INFJ/INFP 玩家的特质（注重内在价值、追求精神共鸣、偏好非暴力冲突、对审美有极高敏感度）。

## 2. 产品功能规划

* **叙事系统**：主线剧情 + 多分支选择，采用“非二元对立”的叙事逻辑。

* **随机事件系统**：设计一套基于 AI 预生成文本的随机事件库，增强冒险的未知感。

* **极简交互系统**：以“剑盾术”为核心的非伤害性互动（安抚与修复）。

* **收集与进度**：萤石碎片的收集、灵脉节点的修复进度，以及基于本地存储的存档机制。

## 3. 视觉与交互 (UI/UX)

* **美术风格**：定义“极简像素治愈风”的具体视觉规范（配色、动效）。

* **交互流程**：针对手机端优化的单手操作逻辑，确保 Web 端加载速度与流畅度。

## 4. 技术概要 (非编程任务)

* **技术选型建议**：前端轻量级框架、静态资源优化策略。

* **数据结构规划**：剧情节点、随机事件库的数据组织形式。
