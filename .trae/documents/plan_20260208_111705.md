我们有两个核心任务来优化体验：

1.  **移除弹框:** 删除 `scripts/main.js` 中的 `alert()` 调用，因为 `ui.js` 已经接管了界面切换逻辑。
2.  **创建独立页面 (`game.html`):**
    *   将目前在 `index.html` 中的 `#game-screen` 及其子元素（插画、剧情、侧边栏）迁移到一个全新的 HTML 文件 `game.html` 中。
    *   `index.html` 将仅作为"主菜单"页面存在。
    *   点击【开始旅途】将不再是简单的隐藏/显示 DOM，而是触发 `window.location.href = 'game.html'` 进行页面跳转。

### 实施步骤

1.  **创建 `game.html`:**
    *   复制 `index.html` 的头部引用（CSS/JS）。
    *   设置 `body` 背景色为 `#222222`。
    *   将 `#game-screen` 结构从 `index.html` 移动到 `game.html` 并设为默认显示。
    *   确保引入 `scripts/game-ui.js` (我们将创建一个新的 UI 脚本专门处理游戏内逻辑，分离主菜单逻辑)。

2.  **清理 `index.html`:**
    *   删除 `#game-screen` 相关的 HTML 结构，只保留主菜单。

3.  **修改脚本逻辑:**
    *   **`scripts/main.js` (在 index.html):** `startNewGame` 函数改为跳转到 `game.html`。
    *   **`scripts/game.js` (新文件，用于 game.html):** 负责初始化游戏界面，绑定选项按钮事件等。
    *   **清理:** 移除旧的 `alert` 弹框。

4.  **样式调整:**
    *   确保 `game.html` 的 `#game-container` 适配深灰色背景，而不是主菜单的封面图。

这样能彻底分离"菜单"与"游戏"的上下文，代码结构更清晰。
